What is AD Manager Pro?
Modern web platform replacing legacy AD tools
  • Web-based interface - No client installation, access from any browser on any device
  • Dark-themed modern UI - Built with React 18 and Tailwind CSS for a professional experience
  • Direct AD integration - Native LDAP/LDAPS via ldap3 library, no external tool dependencies
  • Windows LAPS support - View and rotate local admin passwords with full audit trail
  • Role-based access - Admin, Helpdesk, and Viewer roles with granular permissions
  • Complete audit trail - Every action logged with operator, IP, timestamp, and details
  • Single-port deployment - Backend serves frontend as static files on port 8080
  • Zero-configuration database - SQLite requires no separate DB server
  • Automated installer - PowerShell script handles complete setup in minutes

🎯 What is LAPS?
Windows Local Administrator Password Solution automatically rotates local admin passwords and stores them encrypted in Active Directory. AD Manager Pro provides a secure web interface to view these passwords with mandatory reason logging.

🔒 Security Layers

  • Admin role required - Enforced on both backend (Depends(require_admin)) and frontend
  • Reason validation - Minimum 5 characters required, max 200, checked both sides
  • Every view audited - Success or failure, includes operator, IP, timestamp, reason
  • Password not in listing - Only per-computer fetch endpoint returns password
  • Injection safe - Computer name sanitized to alphanumeric plus hyphens only
  • No caching - Fresh fetch every time, no browser or server cache
  • No bulk export - One password at a time, intentionally limited

Deployment & Setup
One-command automated installation
✨ Automated Installer
The install.ps1 script handles everything: installs Python and Node.js if missing, configures AD, builds frontend, creates Windows scheduled task, opens firewall port, and starts the service. No manual steps required.
# 1. Setup AD permissions on Domain Controller
powershell -ExecutionPolicy Bypass -File setup_permissions.ps1

# 2. Install AD Manager Pro on server
powershell -ExecutionPolicy Bypass -File install.ps1

# 3. (Optional) Enable LAPS integration
Add-ADGroupMember -Identity "IT Admins" -Members "SERVER$"
Restart-Computer -Force

# 4. Access via browser
http://servername:8080
  • Runs on Windows 10/11 or Windows Server 2016+
  • Single port (8080) - no SSL certificate management
  • Auto-start via Windows Task Scheduler as SYSTEM
  • Zero-configuration SQLite database
  • Health check verification post-install
